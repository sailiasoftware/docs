A **course** is the right session type when you're running structured, taught sessions with a fixed start and end. This guide walks through the course setup wizard end to end.

If you're not sure whether you need a course or a hire, see [courses vs hire](/activities/courses-vs-hire) first.

## Step 1: Choose session type

Start a new session and select **Taught course**. This tells Sailia the booking is for an instructor-led programme rather than a time-based rental.

## Step 2: Basic course information

| Field | What to set |
| --- | --- |
| **Course name** | A clear, searchable name (e.g. *RYA Start Windsurf*) |
| **Price per ticket** | The standard price for one participant — you can add more ticket types later |
| **Colour** | Used on the calendar so different courses are easy to spot |
| **Participants per ticket** | How many people each ticket covers (e.g. 2 for a double kayak course) |

## Step 3: Select activities

Activities are how courses are grouped on your booking page. Tick every activity that applies — customers browsing by activity find your course through these tags. Use **Create activity** if a relevant one doesn't exist yet.

## Step 4: Booking settings

| Setting | What it controls |
| --- | --- |
| **Booking method** | Online and phone, or phone-only |
| **Minimum notice period** | How far in advance a customer must book before the course starts |
| **Minimum participants** | The smallest party size accepted on a single booking |
| **Maximum participants** | The largest party size accepted on a single booking |
| **Ask for date of birth** | Whether to collect a DOB during the booking flow |

## Step 5: Resources required

### Qualifications (staff)

Add the qualifications instructors must hold to run the course, along with the instructor-to-participant ratio. For example, *1 RYA Start Windsurf Instructor per 6 participants*. You can add multiple qualifications if different instructors are needed.

### Kit (equipment)

Add the equipment needed and the [kit ratio](/activities/activities#kit-and-resources) — for example, *1 Argo boat per 4 participants*. You can also choose whether each item can be shared across bookings.

### Add-ons (optional extras)

Link any [add-ons](/activities/addon-setup) (lunch, photography, equipment upgrade) that customers can buy alongside the course at checkout.

## Step 6: Connections

| Connection | What it does |
| --- | --- |
| **Waivers** | Documents customers must sign before the course. See [waivers](/customers/waivers). |
| **Communication groups** | Controls which [automated emails](/customers/notifications) fire for this course. |

## Step 7: Course images

Upload high-quality photos. The first image is used in listings; the rest appear in the booking page carousel. Aim for 16:9 landscape orientation — see [uploading images](/getting-started/uploading-images) for full guidance.

## Step 8: Course description

Use the rich text editor to write a clear, scannable description. Include:

- What participants will learn
- What's included in the price
- Prerequisites or fitness requirements
- Duration and schedule

## Step 9: Review and save

The wizard shows a summary before save. Use **Back** to fix anything, then **Save** to create the course.

## What to do next

1. Open **What's On** and add specific dates and times for the course to run.
2. Add additional ticket types (adult, child, member price) with their own pricing.
3. Make a test booking yourself to check the full customer flow.
4. Share your booking page link with customers.
