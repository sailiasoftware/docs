---
title: Terminology updates
description: Recent changes to terminology across the Sailia platform.
---

## Sessions are now activities

The term **Sessions** has been renamed to **Activities** throughout the platform. This change applies to all areas of the UI where you previously saw "sessions," including:

- The **Activities** tab (formerly "Sessions") on the product dashboard
- Activity selectors in booking, discount, instalment plan, and communication workflows
- Activity pass configuration — connected items are now labeled **Connected Activities**
- Gift card restrictions — session-specific gift cards now reference **Activities**
- Waiver editor — the **Activities** tab replaces the former "Sessions" tab
- Financial exports — the "Export planned sessions" option is now **Export planned activities**
- Marketing audience filters — session-based filters now reference **Activities**

<Note>
No configuration changes are required on your part. All existing data, bookings, and settings carry over automatically under the new label.
</Note>

## Activity types are now tags

What was previously called **Activity Types** in settings is now called **Tags**. Tags still serve the same purpose — they categorize your activities for reporting, search, and booking.

You can manage tags from **Settings** the same way you managed activity types before.

## Session groups are now product groups

In the communications workflow editor, **Session Groups** have been renamed to **Product Groups**. This applies to:

- Creating, editing, and deleting groups
- Adding items to groups
- Group labels in workflow trigger displays

The functionality remains the same — only the label has changed.
