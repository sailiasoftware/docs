## Summary
Documents the new kit ratio modes (people per resource vs. resource per person) and clarifies the share-between-bookings setting on activity sessions.

## Changes
- Added a **Kit and resources** section to `activities/activities.mdx` covering kit ratio settings, the two ratio modes, share-between-bookings behaviour, and a step-by-step for adding kit to a session.
- Linked the new section ahead of the existing **Minimum notice period** content so resource configuration sits alongside capacity.

## Context
Triggered by an upstream change that introduces a `RatioMode` setting for session kit, supporting two modes:
- `USERS_PER_KIT` (default) — a fixed number of people share each item of kit.
- `KIT_PER_USER` — each person needs a fixed number of items of kit.

Ratios, kit reservation, and availability calculations now honour the selected mode, and the admin UI exposes it via tabs on the kit configuration popup. The docs update gives operators an end-user explanation of when to use each mode and how it interacts with the existing share-between-bookings option.
